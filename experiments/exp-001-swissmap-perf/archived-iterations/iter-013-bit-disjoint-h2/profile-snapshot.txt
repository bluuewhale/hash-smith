## Current h1/h2 split (iter-011):
- h1 = hash >>> 7          (bits 31..7, 25 bits used for group selection via &(nGroups-1))
- h2 = hash & 0x7F         (bits 6..0, 7 bits fingerprint)
- H2_MASK = 0x0000007F

## Proposed h1/h2 split (iter-013):
- h2 = (byte)(hash >>> 25)  (bits 31..25, 7 bits — MSB=0 guaranteed only if bit31=0)
- h1 = hash & 0x1FFFFFF     (bits 24..0, 25 bits)
- Note: MSB safety — Fibonacci hash is a 32-bit result, bit31 can be 1.
  If bit31=1, (byte)(hash>>>25) = upper 7 bits of int cast to byte;
  EMPTY=0x80, DELETED=0xFE — must ensure h2 != 0x80 and != 0xFE.
  The original design ensures h2 <= 0x7F via mask. With upper-bit h2,
  bit31 maps to sign bit of byte — if hash>>>25 & 0x80 != 0, h2 would collide with EMPTY.
  Fix: use (byte)((hash >>> 25) & 0x7F) to clamp to 7 bits safely.
  This uses bits 31..25 but zeroes bit7 of the byte (the MSB).

## Strategy:
  h2 = (byte)((hash >>> 25) & 0x7F)  — bits 31..25 → 7-bit fingerprint, safe
  h1 = hash & 0x1FFFFFF               — bits 24..0 → 25-bit group index
  These are fully non-overlapping bit ranges.
